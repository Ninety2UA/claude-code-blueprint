## Priority Framework

- **P0**: Must do THIS WEEK — blocking, critical path, or production issue
- **P1**: Important with deadlines — do within 2 weeks
- **P2**: Normal priority — do this quarter
- **P3**: Nice to have — backlog, do when capacity allows

When two tasks have the same priority, prefer the one that:
1. Unblocks other work
2. Reduces risk or uncertainty
3. Has external dependencies (start early to avoid waiting)
4. Is smaller (momentum and quick wins compound)

## Goal Alignment Check

When processing backlog items or prioritizing tasks, cross-reference against these objectives. If a task doesn't support any current goal:
1. Ask whether it should create a new goal
2. Or whether it should be parked in BACKLOG.md with a P3 tag
3. Or whether it should be dropped entirely

Scope creep is the default — resisting it requires active effort.
